# ROADMAP — 核心阶段（写给人看的版本）

> 基础设施已完成（生成 / 编辑 / 导入 / 插件 + 两条轴：设计系统 × 动效配置）。本文件 = **核心功能阶段**的任务拆解。
> 定位（产品方向）：**开源版 Figma Motion** —— 既覆盖它的亮点（追平），又在它够不到的地方取胜（错位）。
> 每个任务都写了：**大白话是什么 · 举个例子 · 解决什么痛点 / 带来什么机遇 · 你会看到什么**。
> 标签：🟦追平 Figma ｜ 🔥我们的差异化。现状与约定见 `PROGRESS.md` / `AGENTS.md`，竞品对照见文件末。

## 一句话定位

- **Figma Motion 打的是**：在设计稿里做动效 → 导出代码/视频交给开发（动的是 Figma 图层，产物是"交接物"，付费，beta）。
- **我们打的是**：**一句话生成 / 编辑"正在跑的真实网页"的动效，带真实交互触发，开源本地，可导出为 skill**。
- 我们的护城河**不是**"Figma 没有触发"（它的原型层有），而是 **"动效活在最终运行的网页里 + 一句话就能改 + 开放可移植"**。

## 现状（已经有了什么）

会的：一句话生成整页 · 对话改整页 · 选风格(设计系统) · 选动效配置 · 点选组件看 Inspector · 5 个"入场"预设 + 调时长/缓动/延迟/位移 ·
底部一条（只能看的）时间轴 · 把库里动效套到组件(只做了入场) · 每个组件的 Motion JSON(只能看/复制) · 改完写回文件 · 导出为 skill / 导入文件 / 导入插件。

**最大短板（一句话）**：现在一个组件**只会一种动法 —— "出现时往上飘一下 + 淡入"**。没有"悬停才动""滚到才出现",也不能转、不能缩放。整个核心都卡在这。

## Milestone A — 「说一句话，组件就在真页面上、带触发地动起来」（先做这个）

> 这是你定的首要"哇"，也是和 Figma 拉开差距的主轴。A1 是地基，后面都靠它。

### A1 ·『动效模型升级』(地基) 🔥 ✅ 完成 (2026-06-26)
- **大白话**：现在每个组件只能记住"一种动法"。升级后，它能记住一整套：**什么时候动、动哪些方面、怎么动、多久**。
- **举例**：一个按钮可以同时存两套动效 ——「出现时淡入」+「鼠标悬停时放大 1.05 倍、轻轻上浮、用弹簧手感、0.3 秒」。
- **痛点**：现在连"悬停放大"都存不下来，组件的动效太单薄。
- **机遇**：这套"组件的动效说明书"就是我们的 **motion-as-JSON** —— 能导出、能让 AI 读和写。它是后面一切的底座。
- **你会看到**：这一步没有独立界面，但做完后 Inspector 里能选触发、能调更多属性（见 A2/A3）。

### A2 ·『触发类型：什么时候动』🔥（Figma Motion 时间轴没有的）✅ 完成 (2026-06-26)
- **大白话**：给动效加一个"什么时候触发"的开关 —— **页面加载时 / 滚动到它时 / 鼠标悬停时 / 点击时**。
- **举例**：「卡片滚动进视线才缓缓浮现」(scroll reveal)、「按钮 hover 才放大发光」、「点赞点一下，爆一串小心心」。
- **痛点**：之前只有"页面一加载就全部播一遍"，做不了真实交互。
- **机遇**：**Figma 的 Motion 时间轴只能"回放"、没有这些触发** —— 这是最一眼能看出我们不一样的地方。
- **你会看到**：Inspector 里多一个「触发」下拉：加载 / 滚动到 / 悬停 / 点击。

### A3 ·『更多能动的方面 + 更多预设』🟦 ✅ 完成 (2026-06-26)
- **大白话**：能动的"方面"从只有"上移+淡入"，扩到 **左右上下移动、放大缩小、旋转、透明度、模糊**；一键预设也变多。
- **举例**：预设里有 淡入 / 四个方向滑入 / 放大入场 / 旋转入场 / 弹一下(pop) —— 和 Figma 的 Fade·Move·Scale·Rotate 对齐，适合 loading、logo reveal、卡片入场。
- **痛点**：只有上移+淡入，套不出常见效果。
- **机遇**：追平 Figma 的预设能力。
- **你会看到**：Inspector 预设区选项更多，可调的滑块更多（位移/缩放/旋转/透明…）。

### A4 ·『弹簧手感 (Spring)』🟦 ✅ 完成 (2026-06-26)
- **大白话**：除了普通的"快慢曲线"，加上"弹簧"那种回弹手感（像 iOS 弹一下）。
- **举例**：按钮按下去弹回来、卡片落定时轻轻晃一下。
- **机遇**：追平 Figma 的 spring。手感是动效产品的灵魂。
- **你会看到**：缓动里多一个「Spring」选项，可调"弹性 / 阻尼"。

### A5 ·『对话 → 组件动效』🔥（你定的 #1「哇」时刻） ✅ 完成 (2026-06-26)
- **大白话**：选中一个组件，在聊天里说「让它悬停时弹一下」，它就**在真页面上动起来**，而且你还能接着说、接着调。
- **举例**：选中标题 → 说「逐字浮现」→ 标题真的逐字浮现；选中按钮 → 说「hover 时放大并发光」→ 立刻生效。
- **痛点**：现在的对话编辑是"把整页丢给 AI 重写"（慢、粗、整页都可能变）；"应用到组件"只是套预设、不能自由描述。
- **机遇**：你定的 **#1 wow**。和 Figma agent 的区别 —— 我们是在**真实在跑的网页上 + 带交互触发 + 结果还能继续编辑**，不是静态稿上摆关键帧。
- **你会看到**：选中组件 → 聊天说一句 → 几秒后它在预览里动起来，Inspector 和 JSON 同步更新。

## Milestone B — 真·可视化时间轴

### B1 ·『能拖能改的时间轴』🟦 ✅ 完成 (2026-06-26)
- **大白话**：现在底部那条时间轴只能看；升级成**能拖能编辑** —— 拖条子改时长/延迟、拖播放头预览任意一刻、点关键帧改数值。
- **举例**：把卡片入场从 0.5 秒拖到 0.8 秒；让第二张卡片比第一张晚 0.1 秒出现（错峰更高级）。
- **痛点**：现在时间轴是装饰，不能真编辑。
- **机遇**：追平 Figma 时间轴，兑现你要的"可视化时间轴"那种交互。

### B2 ·『全页编排 + 克制提醒』🔥 ✅ 完成 (2026-06-26)
- **大白话**：一条总时间轴看到整页所有组件的动效，能排顺序、错峰；要是动效太多太吵，系统提醒"过载了"。
- **举例**：一眼看到"这一屏 12 个东西都在动"，系统提示"一屏最多留一个庆祝动效"。
- **机遇**：把**"克制 / 品味"做成功能** —— 这是我们区别于"又一个素材站"的地方，也是 Figma 不做的。

## Milestone C — 动效系统化（让品牌动效统一）

### C1 ·『命名动效样式 / 动效变量』🟦 ✅ 完成 (2026-06-26)
- **大白话**：把一套动效存成"品牌动效"（比如「我们家的入场」），到处复用；改一处，全站跟着变。
- **举例**：定义一条"品牌缓动曲线"，所有按钮都用它；以后想让全站更弹一点，只改这一个变量。
- **痛点**：现在每个组件各调各的，团队没法统一。
- **机遇**：追平 Figma 的 animated components + motion variables，并接到我们已有的两条轴上，形成"品牌动效规范"。

## Milestone D — 交付与导出（"可交付动效"的开源版）

### D1 ·『导出动效代码』🟦 ✅ 完成 (2026-06-26)
- **大白话**：把做好的动效一键复制成 **CSS / JSON / React** 代码，交给开发直接用。
- **机遇**：追平 Figma 的 Dev Mode 交接；我们产物本来就是真网页，交接损耗更低。

### D2 ·『Lottie 桥接』🔥 ✅ 导出完成·导入待做 (2026-06-26)
- **大白话**：我们的 motion-JSON 能和 **Lottie** 互转 —— 导入别人的 Lottie，也能导出成 Lottie。
- **机遇**：互通能力；Figma 的 Lottie 也有限（原型导 dotLottie、复杂逻辑会被简化），有错位空间。

### D3 ·『视频导出』🟦（拉伸目标，排后面） ⏸ 暂缓（待定）
- **大白话**：把动效录成 MP4 / GIF / WebM 发出去。技术较重（要录屏/无头渲染），不急。

- **E1 ·『时间点评论』**🟦：在时间轴的某一刻留言（追平 Figma；我们已有点元素评论，扩到"时间"维度）。
- **E2 ·『无障碍校验』**🔥：自动检查"有没有给晕动症用户的减弱动效回退"，没有就提醒（品味引擎的一部分，硬规则）。

## 建议的做事顺序（为什么这么排）

1. **A1 → A2 → A3 → A4**：先把"一个组件能拥有丰富的、带触发的动效"这件根本的事做实。没有它，后面全是空中楼阁。
2. **A5**：地基好了，立刻做你要的 #1「哇」—— 一句话让组件动起来。
3. **B1 → B2**：补"可视化时间轴"这种交互 + 立住"克制/品味"的差异化。
4. **C1**：把动效接到两条轴上，形成系统。
5. **D1 → D2 →（D3）**：把"开源版 Figma Motion"的交接闭环补齐。
6. **E**：打磨。

> 🟦 追平 Figma（撑起"开源版 Figma Motion"的可信度）：A3 · A4 · B1 · C1 · D1 · D3 · E1
> 🔥 我们的差异化（真正 motion-anything）：A2 · A5 · B2 · D2 · E2 ＋「动效活在真网页 + 对话即改 + 开放」贯穿始终

## 附：Figma Motion 对照（Config 2026 · open beta · 2026-06-24）

**它有的**：画布内时间轴（预览/调时长/loop·once 等）· 预设(fade/move/scale) · 关键帧+auto-keyframe（位置/缩放/旋转/透明度）· easing + spring ·
AI agent 在时间轴上生成**可编辑**关键帧 · 动效组件 + 动效变量 · 带时间点评论 · Dev Mode 只读时间轴 + 复制 CSS/JSON/React/motion.dev + MCP ·
导出 MP4/GIF/WebM/Animated SVG · shader 属性可 keyframe（3D coming soon）。
**它的边界（= 我们的机会）**：① Motion 时间轴**没有交互触发**（hover/scroll/click 在另一套"原型"里，且复杂条件导出会被简化）；
② 产物是**交接物**（代码/视频/Lottie），动的是 Figma 图层，**不是在跑的、能对话编辑的真实网页**；③ 付费 + 锁定 Figma + beta 不稳；
④ 原生 Motion 今天**不导 Lottie**（走原型→dotLottie / 插件）。
